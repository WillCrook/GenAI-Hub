Lecture Notes to Revision Resources

Outcome: a checked topic summary, active-recall questions and a revision plan based on your own notes.

  1. Organise the source material.
    Do: Gather notes and learning outcomes; remove personal data.
    Prompt/action: Ask the assistant to list the topics it can see without summarising them.
    Expected output: A topic inventory.
    Verify: Compare every topic with the unit materials.
  2. Create a cautious summary.
    Do: Work on one topic at a time.
    Prompt/action: “Summarise only the supplied notes and mark unclear points.”
    Expected output: A concise summary with gaps identified.
    Verify: Correct omissions and unsupported additions.
  3. Build active-recall questions.
    Do: Request questions at mixed difficulty.
    Prompt/action: “Create ten questions; keep answers separate until I respond.”
    Expected output: A question set and answer guide.
    Verify: Remove questions outside the learning outcomes.
  4. Plan and review.
    Do: Rank weak areas and schedule short sessions.
    Prompt/action: “Turn my results into a realistic seven-day plan with breaks.”
    Expected output: An adjustable revision plan.
    Verify: Confirm workload, deadlines and rest time yourself.

Research a Topic Responsibly

Stages: Define the question; search the Library; screen original sources; map claims and evidence; write and check your own synthesis.

Verification: Open every cited source and confirm that it supports the claim attributed to it.

Analyse a Dataset

Stages: Define the analytical question; profile columns and missing values; choose an appropriate method; run reproducible analysis; inspect assumptions and anomalies.

Verification: Re-run calculations independently and retain the code or formula trail.

Improve a CV

Stages: Extract role criteria; map each criterion to genuine evidence; improve clarity; proofread and obtain human feedback.

Verification: Confirm every statement is accurate and sounds like you.

Debug Code

Stages: Reproduce the failure; reduce it to a small example; request a minimal explanation; apply one change and run tests.

Verification: Understand the fix, run existing tests and add a regression test.

Prepare for an Interview

Stages: Analyse the role; select genuine evidence; practise one question at a time; refine clarity without memorising generated prose.

Verification: Keep examples truthful and ask a person to challenge your answers.
